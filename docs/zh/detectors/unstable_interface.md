# 不稳定接口

**ID:** `unstable_interface` | **严重程度:** 中 (默认)

识别那些"移动目标"的模块——当其他人试图在它们之上构建时，它们却不断改变公共 API。

## 为什么这是一种坏味道

- **连锁反应**: 每次你在一个不稳定的模块中改变公共导出时，你可能会破坏依赖它的十几个其他文件。
- **无用功**: 开发者花在修复导入和适应 API 变化上的时间比构建功能还多。
- **令人沮丧**: 很难信任一个每隔一周就违背承诺的模块。

## 如何修复

- **稳定 API**: 在实现之前花更多时间设计接口。
- **使用版本控制**: 如果可能，在过渡期间同时支持多个版本的接口。
- **收窄接口**: 仅导出绝对必要的内容。
