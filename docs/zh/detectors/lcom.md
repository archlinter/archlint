# 低内聚 (LCOM4)

**ID:** `lcom` | **严重程度:** 中 (默认)

内聚性衡量你的类的方法和字段是否真的属于一起。如果不属于，你可能有一个"弗兰肯斯坦怪物"类。

## 为什么这是一种坏味道

- **违反 SRP**: 你的类可能戴着太多帽子，试图同时做三个不同的工作。
- **脆弱性**: 你修改一个与"用户头像"相关的方法，不知怎的破坏了"密码散列"逻辑，因为它们共享同一个臃肿的类。
- **难以复用**: 如果你只需要"头像"逻辑，你被迫也要带上整个"密码"机制。

## 如何修复

1. **Extract Class (提取类)**：将该类拆分为两个或多个更小的类，每个类负责单一职责。
2. **Move Method (移动方法)**：将不使用类状态的方法移动到更合适的位置（例如，工具模块）。

## 配置

```yaml
rules:
  lcom:
    severity: medium
    max_lcom: 4
    min_methods: 3
```
