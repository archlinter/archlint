# Низкая связность (LCOM4)

**ID:** `lcom` | **Степень критичности:** Средняя (по умолчанию)

Связность (cohesion) показывает, действительно ли методы и поля вашего класса подходят друг другу. Если нет — скорее всего, у вас получился класс-Франкенштейн.

## Почему это «запах»

- **Нарушение SRP**: Ваш класс, скорее всего, пытается «сидеть на двух стульях» и делать три разные работы одновременно.
- **Хрупкость**: Вы правите метод для обработки аватарок, а ломаете логику хеширования паролей просто потому, что они живут в одном раздутом классе.
- **Сложность переиспользования**: Если вам нужна только логика для аватарок, вам всё равно придется тащить за собой весь «парольный» механизм.

## Как исправить

1. **Extract Class**: Разделите класс на два или более меньших класса, каждый из которых имеет одну зону ответственности.
2. **Move Method**: Переместите методы, которые не используют состояние класса, в более подходящее место (например, в утилитный модуль).

## Конфигурация

```yaml
rules:
  lcom:
    severity: medium
    max_lcom: 4
    min_methods: 3
    exclude: ['**/*.controller.ts']
```

### Параметры по умолчанию

- `max_lcom`: `4` - максимальное значение метрики LCOM4 (Lack of Cohesion of Methods)
- `min_methods`: `3` - минимальное количество методов в классе для анализа
- `exclude`: `['**/*.controller.ts']` - паттерны файлов, которые исключаются из проверки
