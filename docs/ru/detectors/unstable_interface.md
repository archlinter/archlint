# Неустойчивый интерфейс (Unstable Interface)

**ID:** `unstable_interface` | **Степень критичности:** Средняя (по умолчанию)

Выявляет модули, которые постоянно меняются — они без конца перекраивают свой публичный API, пока все остальные пытаются на этом строить работу.

## Почему это «запах»

- **Эффект домино**: Каждый раз, когда вы меняете экспорт в нестабильном модуле, вы потенциально ломаете дюжину других файлов, которые от него зависят.
- **Бесполезная работа**: Разработчики тратят больше времени на исправление импортов и подстройку под изменения API, чем на создание фич.
- **Потеря доверия**: Трудно полагаться на модуль, который нарушает свои обещания (API) каждую вторую неделю.

## Как исправить

- **Стабилизируйте API**: Уделяйте больше времени проектированию интерфейса перед реализацией.
- **Используйте версионирование**: Если возможно, поддерживайте несколько версий интерфейса одновременно во время переходного периода.
- **Сузьте интерфейс**: Экспортируйте только то, что абсолютно необходимо.

## Конфигурация

```yaml
rules:
  unstable_interface:
    severity: medium
    min_churn: 10
    min_dependents: 5
    score_threshold: 100
```

### Параметры по умолчанию

- `min_churn`: `10` - минимальное количество изменений файла в git (требует `git: { enabled: true }`)
- `min_dependents`: `5` - минимальное количество модулей, зависящих от данного модуля
- `score_threshold`: `100` - пороговое значение нестабильности интерфейса
