# 低凝集 (LCOM4)

**ID:** `lcom` | **重要度:** Medium (default)

凝集度は、クラスのメソッドとフィールドが実際に一緒に属しているかどうかを測定します。もしそうでなければ、おそらく「フランケンシュタインの怪物」クラスがあります。

## なぜこれが「不吉な臭い」なのか

- **SRPの違反**: クラスがおそらく複数の帽子をかぶっていて、同時に3つの異なる仕事をしようとしています。
- **脆さ**: 「ユーザーアバター」に関連するメソッドを変更すると、なぜか「パスワードハッシュ」ロジックが壊れます。なぜなら、同じ肥大化したクラスを共有しているからです。
- **再利用の困難さ**: 「アバター」ロジックだけが必要でも、「パスワード」の仕組み全体も一緒に持ってこなければなりません。

## 修正方法

1. **Extract Class (クラスの抽出)**: クラスを2つ以上の小さなクラスに分割し、それぞれに単一の責任を持たせます。
2. **Move Method (メソッドの移動)**: クラスの状態を使用しないメソッドを、より適切な場所（ユーティリティモジュールなど）に移動します。

## 設定

```yaml
rules:
  lcom:
    severity: medium
    max_lcom: 4
    min_methods: 3
```
