# 不安定なインターフェース

**ID:** `unstable_interface` | **重要度:** Medium (default)

「動く標的」となっているモジュール—他の誰もがその上に構築しようとしているのに、公開APIを常に変更しているモジュールを特定します。

## なぜこれが「不吉な臭い」なのか

- **波及効果**: 不安定なモジュールで公開エクスポートを変更するたびに、それに依存する十数個のファイルを壊す可能性があります。
- **無駄な作業**: 開発者は、機能を構築するよりも、インポートを修正したりAPIの変更に適応したりすることに多くの時間を費やします。
- **フラストレーション**: 2週間ごとに約束を破るモジュールを信頼するのは困難です。

## 修正方法

- **API を安定させる**: 実装前にインターフェースの設計により多くの時間を費やしてください。
- **バージョニングを使用する**: 可能であれば、移行期間中にインターフェースの複数のバージョンを同時にサポートします。
- **インターフェースを狭める**: 絶対に必要なものだけをエクスポートします。
